# Repos-Manager 画面設計書

## UI/UX設計原則

### 1. 情報階層

1. **最重要**: リポジトリ名、主要言語、最終更新日
2. **重要**: Git状態、プロジェクトサイズ、タグ
3. **補助**: 詳細メタデータ、統計情報

### 2. 色彩設計

- **Green** 🟢: 最新・健全・同期済み
- **Yellow** 🟡: 注意・変更あり・中程度
- **Red** 🔴: 警告・エラー・古い・Behind
- **Blue** 🔵: 情報・フォーカス・選択中
- **Gray** ⚫: 非アクティブ・アーカイブ

### 3. アイコン体系

- **📁**: フォルダ/リポジトリ
- **💻**: 言語・技術
- **🔄**: 更新・同期
- **⭐**: お気に入り
- **🏷️**: タグ・分類
- **🔍**: 検索
- **⚙️**: 設定
- **📊**: 統計・分析

### 4. インタラクション設計

- **フォルダ名クリック**: 現在のウィンドウでフォルダを開く
- **⭐/☆ クリック**: お気に入り追加/削除をトグル
- **🪟 ボタンクリック**: 新しいウィンドウでフォルダを開く
- **🔧 ボタンクリック**: クイックアクションメニュー表示（標準・詳細モード）
- **右クリック**: フルコンテキストメニュー表示
- **ホバー**: 行のハイライト＋アクションボタン表示
- **ドラッグ&ドロップ**: 並び替え・グループ化
- **キーボード**: 検索・ナビゲーション

#### 表示モード (デフォルト: コンパクト)

- **コンパクトモード**: 1行表示、最小限の情報、スペース効率重視
- **標準モード**: 2行表示、タグや詳細状態も表示
- **詳細モード**: 多行表示、全ての情報を表示
